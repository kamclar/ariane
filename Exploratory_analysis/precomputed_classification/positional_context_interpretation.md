# What Positional Context Means In The Precomputed Module 1 Map

## Core Statement

Local pathogenic enrichment is a context signal, not an ACMG/ENIGMA evidence
criterion.

This analysis uses a precomputed ARIANE Module 1 coding SNV map. The map was
generated by applying automatable ENIGMA/ACMG Module 1 rules to possible BRCA1
and BRCA2 coding SNVs. It is not a map of biological truth and it is not an
independent clinical classification set.

## What Was Precomputed

The current map contains coding SNVs classified using the automatable criteria
available to ARIANE Module 1 in this snapshot:

- population/frequency criteria: `PM2_Supporting`, `BA1`, `BS1_Supporting`,
  `BS1_Strong`
- predicted or RNA-like splice/computational criteria: `PP3`, `BP4`, `BP7`
- protein consequence criteria: `PVS1`, `PM5_PTC`, `BP1`
- functional or curated evidence criteria available to the module: `PS3`,
  `BS3`, `PS1`, `PP4`, `BP5`

Non-automated evidence such as segregation, case-control data, trans data,
manual expert curation, and some functional/RNA interpretation remains outside
this map.

## Position Is Not Variant

A coding position is only a coordinate. A variant is a specific nucleotide
change at that coordinate.

At one position, different substitutions can have different consequences:

- synonymous change
- missense change
- premature termination codon
- splice motif disruption
- different functional evidence or curated evidence context

Therefore, a position can contain both benign and pathogenic generated variants.
The same can happen within one codon.

This is not automatically a contradiction. It means that the exact nucleotide
change and exact consequence matter more than the coordinate label.

## Region Is Not Evidence

A region can be enriched for generated class 4/5 variants for several reasons:

- many possible substitutions create premature stop codons
- nearby bases are splice-sensitive
- the region contains functional assay evidence
- the region contains known domain or residue context
- the current automated rules create a local pattern

That enrichment is useful because it points to places where manual review may
be informative. It does not mean that every VUS in the region is pathogenic.

## How To Use Local Context

Use local pathogenic enrichment as:

- a review-priority signal
- a prompt to inspect variant mechanism
- a prompt to inspect nearby benign counterexamples
- a prompt to check whether splice, truncation, functional evidence, or domain
  context drives the local signal

Do not use local pathogenic enrichment as:

- an ACMG/ENIGMA criterion
- a replacement for accepted evidence such as `PS1`, `PS3`, `PP3`, `PVS1`,
  `PM5_PTC`, `PP1`, `PM3`, `PS4`, or `BS2`
- a standalone pathogenicity prediction
- a reason to reclassify a VUS automatically

## Practical Interpretation

The right interpretation is:

> This VUS lies in a local context where the precomputed automated Module 1 map
> contains more pathogenic-direction generated variants than expected.

The wrong interpretation is:

> This VUS is pathogenic because it is near pathogenic variants.

## Example: BRCA1 BRCT1 Mixed Cluster

The BRCA1 `c.5044-c.5143` region, approximately `p.Glu1682-p.Ser1715`, is a
clear example. It lies in the BRCT1 subdomain and appears as a benign-enriched
VUS cluster in the +/-20 bp local-neighborhood plot.

The region is not simply benign. In the precomputed map it contains:

- generated class 1/2 variants
- VUS
- generated class 4/5 variants

Its mixed behavior appears to reflect a combination of BRCT functional evidence,
`BS3`/`PS3`, local `PP3`/SpliceAI signal near a donor-like boundary, and nearby
truncating substitutions.

This makes it useful for manual review and explanation, but not as an
independent classification rule.
