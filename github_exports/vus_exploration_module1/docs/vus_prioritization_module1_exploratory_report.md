# BRCA1/2 VUS Prioritization From A Precomputed ARIANE Module 1 Variant Map

## Short Summary

This exploratory report describes a working framework for selecting BRCA1/2 VUS
candidates that deserve earlier manual review. The framework does not predict
pathogenicity and does not reclassify variants. It uses a precomputed ARIANE
Module 1 coding SNV map, generated by automatable ENIGMA/ACMG rules, to
identify VUS with stronger review signals: SpliceAI signal, proximity to exon
boundaries, local pathogenic density, evidence combinations that nearly satisfy
ENIGMA rules, structural/domain context, and evidence conflicts such as
`BS3+PP3`.

The output is a ranked curation queue, not a clinical classification.

## Background

Variants of uncertain significance remain a major practical problem in BRCA1/2
interpretation. Even when a rule-based framework is available, only part of the
ACMG/ENIGMA evidence set can be automated reliably. In our ARIANE application,
the current automated part corresponds to Module 1: criteria that can be applied
from structured inputs and precomputed resources.

Some functional, curated, or multifactorial-likelihood criteria can appear in
this generated Module 1 map when they are available as structured local
resources. In this snapshot, `PS3`, `BS3`, and `PS1` represent structured
functional or same-amino-acid evidence lookups. `PP4` and `BP5` should be
treated separately: in the ENIGMA context they represent precomputed
multifactorial likelihood-ratio evidence, not functional assay evidence and not
the generic ACMG BP5 co-observation rule. This does not mean that all
functional, RNA, phenotype, family, or clinical evidence is automatically
interpreted. Evidence that is not available in a structured and rule-compatible
form still requires manual curation, including segregation, case-control data,
trans observations, curated clinical review, and many functional/RNA
interpretations.

Therefore, the precomputed automated Module 1 map naturally contains a large
and heterogeneous VUS set. These VUS are not equivalent. Some have little
information beyond population absence. Others are close to a likely pathogenic
or likely benign combination but are missing one accepted evidence item. A
practical curation tool should help identify which VUS are worth opening first.

## Aim

The aim was to build an exploratory triage framework for BRCA1/2 VUS candidates
from the precomputed ARIANE Module 1 variant map.

The framework asks:

- Which VUS have evidence patterns suggesting that manual review may be
  informative?
- Which VUS are located in neighborhoods enriched for generated pathogenic
  variants?
- Which VUS are closer to generated pathogenic than benign variants?
- Which VUS fall in functionally important protein regions?
- Which apparent signals are weakened by benign counterexamples or conflicting
  evidence?

The framework explicitly does not ask:

- Is this VUS pathogenic?
- Should the VUS be upgraded automatically?
- Can domain membership be used as an ACMG/ENIGMA criterion?

## Practical Value And Intended Use

The practical purpose of this framework is to reduce the first-pass manual
review burden. It separates low-information VUS from variants where additional
evidence review is more likely to be informative, and it labels why each VUS is
being prioritized.

The expected value is:

- better ordering of curator effort across thousands of Module 1 VUS
- clearer labels for why a VUS is in the review queue
- early detection of evidence conflicts, such as `BS3+PP3` or public assertion
  discordance
- separation of variant-level signals from neighborhood-driven context
- explicit identification of variants that are close to a likely pathogenic or
  likely benign evidence combination

The framework is therefore an audit and triage layer. It is not intended to
produce new clinical classifications by itself.

## Multiple Prioritization Queues

VUS prioritization is not a single ranking problem. Different variants should
be reviewed early for different reasons, and these reasons should not be mixed
up with ACMG/ENIGMA evidence criteria. In this report, prioritization is treated
as a set of review queues that help decide what a curator should open first.

The main prioritization queues are:

- near-threshold variants: VUS that are one accepted evidence item away from a
  likely pathogenic or likely benign Module 1 combination
- splice/RNA review candidates: VUS with high SpliceAI signal, exon-boundary
  proximity, or a pattern where RNA evidence could resolve the uncertainty
- functional-evidence candidates: VUS with structured functional data, or VUS
  in regions where assay metadata and assay mechanism need careful review
- evidence-conflict candidates: VUS with opposing signals, such as functional
  benign evidence together with computational splice or protein-impact signal
- public-assertion candidates: VUS with ClinVar, ClinGen/ENIGMA assertion,
  submitter-discordance, or panel-level context that should be compared with
  the local Module 1 result
- neighborhood/context candidates: VUS located in regions enriched for
  generated pathogenic or benign Module 1 classifications
- low-information variants: usually PM2-only or otherwise weakly annotated VUS,
  where manual review may be lower yield unless external clinical, family,
  RNA, or functional data exist

These queues can overlap. For example, a VUS may be both near a likely
pathogenic threshold and located in a pathogenic neighborhood, or it may have a
high SpliceAI score but also public benign assertions. The review label should
therefore explain the reason for prioritization, not only the numeric rank.

This is also why local pathogenic enrichment is used only as a context signal.
It can make a VUS more interesting for manual review, but it cannot add points
or replace accepted variant-level ACMG/ENIGMA evidence.

This concept was also implemented as a separate queue-synthesis analysis. Each
VUS was assigned to zero or more practical review queues: near threshold,
splice/RNA, functional, evidence conflict, public assertion, neighborhood
context, and low information. In the current snapshot, the largest queues were
neighborhood context (`6649` VUS), low information (`5379` VUS), and splice/RNA
(`3291` VUS). Smaller but more actionable queues included functional evidence
(`836` VUS), near-threshold variants (`637` VUS), public assertions (`72` VUS),
and explicit evidence conflicts (`30` VUS).

Most VUS belonged to more than one queue. This supports the core interpretation
that a practical VUS worklist should be filtered by review question, not only
sorted by one global priority score.

## Gene-Normalized Comparisons

Several plots and summary tables initially combined BRCA1 and BRCA2. This is
useful for a first overview, but raw combined counts can be misleading because
BRCA2 contributes more generated coding SNVs than BRCA1 in the current
snapshot. Cross-gene comparisons should therefore use within-gene percentages
or rates per 1000 generated variants, and VUS worklist comparisons should use
rates per 1000 VUS in each gene.

A gene-normalized layer was added for the combined analyses. In the current
snapshot:

- BRCA1 contributes `16776` generated coding SNVs and `1625` VUS.
- BRCA2 contributes `30771` generated coding SNVs and `6529` VUS.
- Generated pathogenic classes 4/5 are similar after normalization: BRCA1
  `52.28` per 1000 generated SNVs, BRCA2 `50.83` per 1000 generated SNVs.
- VUS are much more frequent in BRCA2 after normalization: BRCA1 `96.86` per
  1000 generated SNVs, BRCA2 `212.18` per 1000 generated SNVs.
- Among VUS, near-threshold queue membership is much more frequent in BRCA1:
  BRCA1 `288.00` per 1000 VUS, BRCA2 `25.88` per 1000 VUS.
- Among VUS, functional-queue membership is also much more frequent in BRCA1:
  BRCA1 `420.92` per 1000 VUS, BRCA2 `23.28` per 1000 VUS. This likely
  reflects available structured BRCA1 functional evidence, not necessarily a
  true biological difference.
- Low-information VUS are more frequent in BRCA2: BRCA1 `478.15` per 1000
  VUS, BRCA2 `704.86` per 1000 VUS.

The normalized results change how the combined analyses should be read. For
example, a combined VUS queue can look large because BRCA2 contributes more
VUS, while actionability signals such as structured functional evidence and
near-threshold combinations are concentrated in BRCA1 in this snapshot.

## Dataset Generation

The starting point was the full precomputed BRCA1/2 coding SNV classification
map:

`variant_space_scan/outputs/brca_module1_full_snv_classification.csv`

This map was generated by applying the current ARIANE automatable Module 1
rules to coding SNVs in BRCA1 and BRCA2. In the current snapshot, the observed
automated criteria were:

- population/frequency criteria: `PM2_Supporting`, `BA1`, `BS1_Supporting`,
  `BS1_Strong`
- predicted or RNA-like splice/computational criteria: `PP3`, `BP4`, `BP7`
- protein consequence criteria: `PVS1`, `PM5_PTC`, `BP1`
- structured functional or curated evidence criteria available to this local
  snapshot: `PS3`, `BS3`, `PS1`
- precomputed multifactorial likelihood-ratio lookup criteria available to
  this local snapshot: `PP4`, `BP5`

Two implementation labels require special caution. `PM5_PTC` is not a base
ACMG code name, but it is an ENIGMA-style exon-specific
premature-termination-codon weighting used for PTC contexts supported by
functional, case-control, or family data. `PP4` and `BP5` are included because
they appear in the current structured snapshot as precomputed multifactorial LR
lookups; they should not be interpreted as newly computed phenotype or
co-observation criteria by ARIANE.

The map includes:

- gene and HGVS c./p. notation
- variant type
- SpliceAI score from the local SpliceAI cache
- population-frequency status from the local gnomAD cache
- applied automated ACMG/ENIGMA criteria
- total point score
- generated class 1 to 5 output from those automatable rules
- warnings and notes about non-automated criteria

The analysis was limited to coding SNVs. It does not include indels,
multi-exon CNVs, deep intronic variants, promoter variants, UTR variants, or
other noncoding variant classes.

## Important Interpretation Boundary

All analyses in this report describe patterns in the precomputed ARIANE Module
1 variant map. This is not a map of biological reality, and it is not an
independent clinical reference set. It is a map of how coding SNVs are
classified when the current automatable Module 1 rules are applied.

Terms such as pathogenic density, pathogenic neighborhood, domain enrichment,
or high-priority VUS mean that a variant is interesting for manual review within
this precomputed automated-rule map. They do not mean that the variant is
pathogenic.

Local pathogenic enrichment is a context signal, not an ACMG/ENIGMA evidence
criterion.

## Methods

### VUS Selection

Variants classified as class 3 by the automated Module 1 snapshot were selected
as VUS. In the current snapshot, the VUS set contained:

- total VUS: `8154`

Each VUS was annotated with:

- CDS position
- CDS exon
- distance to the nearest CDS exon boundary
- SpliceAI score
- applied criteria
- local counts of nearby benign, VUS, and pathogenic generated variants
- review category
- priority score

### Review Priority Score

A heuristic review score was assigned to each VUS. The score is not a classifier.
It is a practical sorting key for manual review.

Signals that increase priority are additive, but the SpliceAI and boundary
components are tiered rather than double-counted. A variant with SpliceAI >=
0.20 receives the 30-point high-SpliceAI weight, not both the 30-point and
12-point weights. A variant within 2 bp of a CDS exon boundary receives the
20-point boundary weight, not both the 20-point and 8-point weights.

The current heuristic weights are:

| Priority component | Weight |
| --- | ---: |
| SpliceAI >= 0.20 | 30 |
| SpliceAI >= 0.10 and < 0.20 | 12 |
| within 2 bp of a CDS exon boundary | 20 |
| within 10 bp but >2 bp from a CDS exon boundary | 8 |
| `PVS1` present | 35 |
| `PS3` present | 25 |
| `PS1` present | 18 |
| `PP3` present | 15 |
| `PP4` present as precomputed multifactorial LR evidence | 15 |
| `BS3` present | 8 |
| `near_pathogenic_dense`: at least 5 class 4/5 variants within +/-20 coding bp | 12 |
| `high_spliceai_in_benign_dense_region`: SpliceAI >= 0.20 and at least 20 class 1/2 variants within +/-20 coding bp | 8 |

The tiers are assigned from the total score: `tier1_urgent` for score >= 80,
`tier2_high` for score >= 50, `tier3_medium` for score >= 25, and `tier4_low`
below 25.

SpliceAI >= 0.20 is used here only as a sensitive review-triage threshold. It
is intentionally permissive and may over-call possible splice effects; it is
not used here as a standalone classification threshold.

`PP4` is included in the triage score because a VUS with precomputed
pathogenic-direction multifactorial LR evidence is useful to review early.
This does not mean ARIANE calculated new phenotype or family-history evidence;
it means the variant already carries a structured curated LR lookup.

The resulting tiers were:

| Tier | Count |
| --- | ---: |
| tier1_urgent | 99 |
| tier2_high | 760 |
| tier3_medium | 733 |
| tier4_low | 6562 |

The practical first-pass review list is therefore not all VUS, but primarily
the `tier1_urgent` and `tier2_high` groups.

### Neighborhood Contribution To The Priority Score

Because the precomputed map is generated by the same automated Module 1 rules
that define the neighboring benign and pathogenic groups, local-neighborhood
signals are not independent evidence. We therefore repeated the priority-tier
assignment after removing the two neighborhood-derived score components:

- `near_pathogenic_dense`: 12 points
- `high_spliceai_in_benign_dense_region`: 8 points

This sensitivity analysis showed that neighborhood context contributes
substantially to the current high-priority queue:

| Metric | Count | Percent of VUS |
| --- | ---: | ---: |
| total VUS | 8154 | 100.00 |
| VUS with any neighborhood reason | 6249 | 76.64 |
| VUS with only neighborhood reasons | 5018 | 61.54 |
| tier1/tier2 original | 859 | 10.53 |
| tier1/tier2 without neighborhood points | 286 | 3.51 |
| lost tier1/tier2 without neighborhood points | 573 | 7.03 |

The interpretation is therefore deliberately cautious. VUS that remain
tier1/tier2 after removing neighborhood points are prioritized mainly by
variant-level signals such as SpliceAI, boundary proximity, `PS3`, `PP3`,
`PS1`, `PVS1`, or `BS3`, or by precomputed multifactorial LR evidence such as
`PP4`. VUS that drop out of tier1/tier2 after removing neighborhood points
should be labelled as neighborhood-driven candidates. These are useful for
mechanism review and visual triage, but they should not be presented as
independently high-priority without external validation.

### VUS Bottleneck Analysis

VUS were grouped by the reason they remained uncertain. The major bottlenecks
were:

| Bottleneck | Count | Interpretation |
| --- | ---: | --- |
| PM2_only | 5046 | population absence is the only supporting pathogenic evidence |
| mixed_splice_benign_and_pathogenic | 1276 | benign splice evidence plus PM2 background |
| computational_pathogenic_evidence_not_enough | 842 | PP3 or SpliceAI signal is insufficient alone |
| strong_pathogenic_combo_one_step_short | 446 | strong pathogenic evidence but missing one accepted item |
| no_automated_evidence | 331 | no useful Module 1 evidence |
| benign_functional_evidence_not_enough | 170 | benign evidence present but not enough for benign class |

This short table shows the largest bottleneck groups, not all mutually
exclusive categories. The complete bottleneck analysis contains 8154 VUS; the

This shows that most VUS are not dramatic edge cases. Many are low-information
variants where automated Module 1 evidence is incomplete. `PM2_Supporting` is
an independent supporting pathogenic criterion, but it is weak and insufficient
by itself to move a variant out of VUS.

### Evidence Action Plan

The bottleneck table was converted into a practical next-evidence worklist.
This does not add points and does not reclassify variants. It asks which type
of additional variant-level evidence would be most useful for each unresolved
VUS group.

| Action group | Count | Main interpretation |
| --- | ---: | --- |
| background_absence_only | 5048 | PM2 alone or weak context only; low first-pass yield unless external evidence already exists |
| conflicting_computational_splice_context | 1276 | benign splice/RNA branch and pathogenic context need transcript-aware manual review |
| computational_signal_needs_noncomputational_support | 842 | SpliceAI/PP3 signal needs RNA, functional, PS4, PP1, PM3, or curated assertion support |
| near_pathogenic_threshold | 446 | one independent pathogenic evidence item may move the case toward likely pathogenic |
| module1_no_signal | 331 | no useful automated Module 1 signal; external public/literature/lab evidence is needed |
| near_benign_threshold | 191 | extra benign evidence or stronger benign functional evidence may resolve the case |
| manual_conflict_adjudication | 20 | mixed benign and pathogenic evidence requires case-specific review |

This makes the review queue more useful than a simple ranked list. Variants
whose only pathogenic-direction signal is a computational splice prediction are
good targets for RNA or functional follow-up. Variants with nothing beyond
population absence are usually not efficient first-pass targets unless public,
literature, or local laboratory evidence already exists.

### Independent Functional Axis: Findlay SGE

For the one setting with dense functional data, the BRCA1 RING and BRCT
regions, the priority score can be checked against a continuous saturation
genome editing score from Findlay et al. 2018. The check is only partial. There
is no comparable dense axis for BRCA2, and even here the independence is
imperfect because some local `PS3` and `BS3` evidence comes from the same SGE
dataset. With that caveat, 673 BRCA1 VUS could be matched to an SGE score.

| Priority tier | Count | Median function score | LOF | INT | FUNC |
| --- | ---: | ---: | ---: | ---: | ---: |
| tier1_urgent | 9 | -1.951 | 6 | 3 | 0 |
| tier2_high | 46 | -1.647 | 30 | 15 | 1 |
| tier3_medium | 365 | -1.891 | 303 | 15 | 47 |
| tier4_low | 253 | -0.828 | 14 | 139 | 100 |

The agreement is only coarse. Tier4 stands apart as the more
function-preserved, benign-leaning group, so the score separates out many cases
least worth reviewing first. The finer ordering is not supported. Tier3, not
tier1 or tier2, contains the largest number and fraction of LOF-class variants:
303 of 365 tier3 VUS are LOF, compared with 6 of 9 in tier1 and 30 of 46 in
tier2. The tier1 and tier2 counts are small, so their medians should not be
overread.

This inversion is the informative part. Almost all of those tier3 LOF variants,
302 of 303, already carry `PS3` in the combination `PM2_Supporting+PS3`. They
sit in tier3 not because the evidence is weak, but because of how the review
score is built. `PS3` is worth 25 priority points, and tier3 begins at 25, so a
variant whose strongest signal is measured functional damage lands at the floor
of tier3 by construction. A high SpliceAI prediction scores 30 and reaches a
higher tier. As currently weighted, the review score can place measured loss of
function below a predicted splice effect.

This points to a concrete sensitivity test for a later version: whether direct
functional evidence should carry at least as much triage weight as a prediction,
with review-priority weighting kept separate from classification scoring.

The same finding appears from the evidence-action perspective:

| Tier3 Findlay LOF overlap | Count | Dominant criteria combo |
| --- | ---: | --- |
| `near_pathogenic_threshold` / `strong_pathogenic_combo_one_step_short` | 302 | `PM2_Supporting+PS3` |
| `background_absence_only` / `PM2_plus_weak_context` | 1 | `PM2_Supporting+PP4` |

Thus, Findlay SGE, the evidence action plan, and the Group 1 bottleneck analysis
describe the same practical queue from different angles. Review priority and
proximity to likely pathogenic are correlated, but they are not the same
measurement. These 302 variants are the main place where the two disagree.

### Does Positional Context Add Independent Information?

The positional analyses answer one central question: does local pathogenic
enrichment add information beyond the variant-level criteria already present in
Module 1?

The short answer is mostly no. Local context is useful for prioritization,
visual triage, and mechanism review, but it should not be treated as an
independent evidence criterion. After decomposing truncation, splice/PP3, and
functional-evidence drivers, the apparent regional pathogenic signal is largely
explained by variant-level criteria already present in the map.

This is still useful. It tells the curator what kind of mechanism dominates
near a VUS: truncation, splice, functional evidence, or a mixture. The result is
a prompt for review, not a new source of ACMG/ENIGMA points.

### Pathogenic Neighborhood And Proximity

The local coding sequence neighborhood around each VUS was compared with the
generated benign and pathogenic groups:

- benign group: classes 1 and 2
- pathogenic group: classes 4 and 5
- VUS group: class 3

For each VUS, the analysis computed:

- nearest generated pathogenic variant
- nearest generated benign variant
- counts of pathogenic and benign variants within +/-20, +/-50, and +/-100 bp
- whether the VUS lies in a pathogenic hotspot bin

In this analysis, a broad pathogenic context means that at least one of the
following is true: the VUS lies in a pathogenic hotspot bin; at least 5 class
4/5 variants occur within +/-20 coding bp; at least 10 class 4/5 variants occur
within +/-50 coding bp; or at least 20 class 4/5 variants occur within +/-100
coding bp. A strong pathogenic region is stricter: pathogenic hotspot bin, or
at least 10 class 4/5 variants within +/-20 coding bp with more pathogenic than
benign neighbors, or at least 20 class 4/5 variants within +/-50 coding bp with
at least 1.5x as many pathogenic as benign neighbors.

The +/-50 bp component is retained in the numerical definition of strong
pathogenic region, but the standalone +/-50 bp scatter plot was removed from
the report to keep the visual interpretation focused on the stricter +/-20 bp
local view.

Results:

| Category | Count |
| --- | ---: |
| closer_to_benign | 3853 |
| closer_to_pathogenic | 3480 |
| equal_distance | 821 |
| strong_pathogenic_region | 2301 |
| strong_pathogenic_region_and_closer_to_pathogenic | 1451 |

The `1451` VUS in strong pathogenic regions and closer to pathogenic than benign
variants form an important manual-review candidate group. This still does not
imply pathogenicity; it identifies VUS located in locally pathogenic-looking
generated landscapes.

One visible cluster in the +/-20 bp neighborhood plot maps to BRCA1
`c.5044-c.5143`, approximately `p.Glu1682-p.Ser1715`, within the BRCT1
subdomain of the BRCA1 C-terminal BRCT region. This cluster is benign-enriched
in the generated local-neighborhood plot, but it is not a benign region. The
full generated SNV set in this short interval contains class 1/2, VUS, and
class 4/5 variants. The pattern appears to be driven by a mixture of functional
BRCT evidence (`BS3`/`PS3`), some local `PP3`/SpliceAI signal near a donor-like
boundary around `c.5074`, and nearby nonsense substitutions with
truncation-related criteria. This makes it a useful example of why local
neighborhood context must be interpreted together with the exact variant
consequence.

### What Positional Context Means

The local-neighborhood analyses ask what positional context means in this
precomputed automated-rule map. They do not ask whether position alone can
classify a variant.

A gene position is not the same as a variant. A single coding position can have
several possible nucleotide substitutions. One substitution may be synonymous,
another may be missense, another may create a premature termination codon, and
another may alter a splice motif. Therefore, variants at the same position, or
even in the same codon, can have different generated classes.

This was confirmed by the position/codon conflict analysis. Many coding
positions and codons contain different possible SNVs with different generated
grouped classes. This is expected in part, because truncating, missense,
synonymous, splice-affecting, and functionally annotated substitutions can
activate different Module 1 criteria.

The same principle applies to local neighborhoods. A region can be enriched for
generated class 4/5 variants because it contains many truncating possibilities,
splice-sensitive bases, known functional evidence, or curated residue/context
signals. A nearby VUS does not inherit that evidence just because it is close.
For the VUS itself, the exact nucleotide change, protein consequence, SpliceAI
score, functional evidence, population-frequency evidence, and accepted
ACMG/ENIGMA criteria still have to be evaluated.

Therefore, local pathogenic enrichment should be used as:

- a review-priority signal
- a prompt to inspect mechanism and nearby counterexamples
- a way to find mixed regions where the automated rules behave differently
  across adjacent variants

It should not be used as:

- a standalone pathogenicity prediction
- a replacement for `PS1`, `PS3`, `PP3`, `PVS1`, `PM5_PTC`, or any other
  criterion
- an ACMG/ENIGMA evidence item
- a reason to upgrade a VUS automatically

In short: local pathogenic enrichment is a context signal, not an ACMG/ENIGMA
evidence criterion.

### Regional Driver Decomposition

To understand why some regions look pathogenic-enriched in the precomputed
Module 1 map, we decomposed each 100 bp CDS bin into candidate drivers:

- truncation-related signal: nonsense variants, `PVS1`, or `PM5_PTC`
- splice-related signal: `PP3` or SpliceAI >= 0.20
- pathogenic functional signal: `PS3` or `PS1`
- pathogenic multifactorial LR signal: `PP4`
- benign functional signal: `BS3`
- benign rule or multifactorial LR signal: `BP1`, `BP4`, `BP5`, `BP7`, `BA1`,
  or `BS1`
- low-information signal: no criterion or `PM2_Supporting` only

The analysis then recalculated regional pathogenic fraction after excluding
major driver classes. In the current 100 bp bin analysis, regional pathogenic
fraction dropped strongly after excluding truncation drivers. After excluding
truncation, splice, and functional-evidence drivers together, the residual
pathogenic fraction was zero across the analyzed bins.

This result should be interpreted as a structural sanity check of the rule
system, not as an independent empirical discovery. In the current automated
Module 1 map, class 4/5 outcomes are generated mainly by exactly these
pathogenic driver classes. Once they are removed, `PM2_Supporting` remains too
weak to generate a class 4/5 result by itself. The useful conclusion is
therefore not that biology has no residual regional signal, but that this
precomputed map should not be read as evidence for an independent positional
criterion.

### Positional Context Follow-up

We then tested the same interpretation from four additional angles:

- variant type
- splice-driver regional signal
- functional-evidence regional signal
- mixed position and codon behavior after removing major driver classes

Variant type had a strong effect on the generated map. Nonsense/PTC variants
were overwhelmingly classified as pathogenic in the automated snapshot, while
missense and synonymous variants were mostly benign or VUS. This is expected,
because the exact substitution determines whether a variant activates
truncation-related, splice-related, functional, benign, or low-information
criteria.

The splice-driver analysis identified regional bins where local pathogenic
fraction tracked `PP3`, high SpliceAI, or proximity to CDS exon boundaries.
The functional-evidence analysis highlighted BRCA1 RING and BRCT regions,
where `PS3` and `BS3` evidence both occur. These regions are biologically and
curationally interesting, but the signal still comes from variant-level
evidence rather than from coordinate alone.

`PP4` was handled as a separate multifactorial LR lookup signal, not as
functional evidence.

Finally, we repeated the mixed position/codon analysis after excluding major
driver classes. In the full generated map, 1654 coding positions and 1737
codons contained both benign and pathogenic grouped classes. After excluding
truncation, splice, and functional-evidence drivers together, this number fell
to zero for both positions and codons. This is expected from the construction
of the current rule set, because the excluded drivers are also the main ways a
coding SNV becomes class 4/5 in this map. The analysis is still useful as a
guardrail: it shows why position-level summaries must be decomposed back to the
variant-level criteria that generated them.

## Candidate Review Groups

The analyses suggest several candidate groups for manual review.

### Group 1: Strong Pathogenic Evidence One Step Short

These VUS often contain `PS3`, `PP3`, or another pathogenic-direction signal but
do not satisfy the exact ENIGMA combination needed for likely pathogenic.
Similarly, a variant can carry `PVS1` and still remain VUS if the applicable
PVS1 strength or the full ENIGMA combination table does not meet a likely
pathogenic combination without additional supporting evidence.

Typical next evidence to look for:

- RNA confirmation
- accepted functional evidence
- `PS1`
- `PP1`
- `PM3`
- `PS4`
- `PP4`
- curated external evidence

### Group 2: VUS In Strong Pathogenic Regions

These VUS are in generated regions where class 4/5 variants outnumber benign
variants locally, or where the nearest generated pathogenic variant is closer
than the nearest generated benign variant.

Useful review questions:

- Is the local signal driven by splice effects?
- Is the VUS near an exon boundary?
- Does the VUS have high SpliceAI?
- Does the same codon or residue have curated pathogenic substitutions?
- Are there benign counterexamples nearby?

### Group 3: Functional-Domain VUS With Supporting Evidence

Domain or interface context can help prioritize VUS, especially when combined
with other signals.

However, domain context is not evidence by itself. Benign counterexamples in
RING, BRCT, BRCA2 DBD, PALB2, RAD51/BRC, and SEM1/DSS1 contexts show why domain
membership must be interpreted cautiously.

### Group 4: BS3 Conflict Cases

Variants with `BS3` plus `PP3` or high SpliceAI should be reviewed as
evidence-adjudication cases.

Useful review questions:

- What did the BS3 assay measure?
- Does the assay detect splicing?
- Does the assay detect NMD?
- Is the PP3 signal transcript-relevant?
- Is the apparent conflict protein-based, RNA-based, or rule-based?

### Group 5: Low-Information PM2-Only VUS

The largest group is `PM2_only`. These variants are not good first-pass review
candidates unless additional evidence is available. Population absence as
`PM2_Supporting` is an independent supporting pathogenic criterion, but it is
not sufficient by itself.

## Proposed Figure Plan

For a reader-facing manuscript, the main figures should emphasize synthesis
rather than every exploratory diagnostic plot. Conceptual and decision-facing
figures should stay in the main text. Dense diagnostic tracks and exploratory
scatter plots can move to a supplement unless they are accompanied by a clear
take-home message.

### How To Read The Main Graphs

The figures in this report summarize behavior of the generated ARIANE Module 1
map. They should be read as explanations of rule output, not as independent
clinical reference evidence.

In bar charts, the bar length is the number or fraction of generated variants
in the stated category. Colors group related classes or driver types; they do
not add evidence by themselves.

In local-neighborhood scatter plots, each dot is one VUS. The VUS itself is
the center of the window. The x-axis counts generated benign/likely benign
class 1/2 variants around that VUS. The y-axis counts generated likely
pathogenic/pathogenic class 4/5 variants around the same VUS. A dot further to
the right has more benign neighbors; a dot higher up has more pathogenic
neighbors. Red dots are pathogenic-neighbor enriched, green dots are
benign-neighbor enriched, grey dots have equal counts, and a black outline
marks a strong context flag. This is a review-context signal only.

In regional driver plots, the x-axis follows coding-sequence position or
100 bp CDS bins. Driver colors indicate which variant-level rule families
explain the local signal, such as truncation, splice/PP3, or functional
evidence. These plots are meant to decompose the source of enrichment, not to
define new region-level ACMG/ENIGMA criteria.

In ablation plots, rows or bars show what remains after removing one or more
driver classes. If a signal disappears after removing a driver, this means the
signal was generated by that rule family in the current map. It should not be
read as an independent biological validation.

## Practical Curation Workflow

1. Start with `tier1_urgent` and `tier2_high` VUS, but also pull the
   near-pathogenic-threshold group even when it sits in tier3. This especially
   applies to `PM2_Supporting+PS3` variants: the Findlay SGE analysis shows
   that many are functionally LOF and one evidence item short.
2. Check whether the VUS belongs to a clear bottleneck category.
3. Separate "high review priority" from "close to likely pathogenic"; these are
   related, but the PS3-driven tier3 group shows that they are not identical.
4. If splice signal is present, verify transcript relevance and boundary
   context.
5. If the VUS lies in a pathogenic region, inspect nearby benign and pathogenic
   variants.
6. If the VUS lies in RING, BRCT, BRCA2 DBD, PALB2, RAD51/BRC, or SEM1/DSS1
   context, inspect benign counterexamples and residue-level annotations.
7. If `BS3` conflicts with `PP3` or high SpliceAI, inspect assay metadata before
   interpreting the conflict.
8. Search for missing accepted evidence: RNA, functional assay metadata, PS1,
   PP1, PM3, PS4, PP4, or curated ClinGen/ENIGMA and public laboratory
   assertion context.

## Limitations

This analysis has several important limitations:

- It studies a generated ARIANE Module 1 snapshot, not an independent clinical
  reference set.
- It does not reclassify variants.
- It covers coding SNVs only.
- It does not include indels, CNVs, deep intronic variants, promoter variants,
  or UTR variants.
- It uses only currently automated Module 1 criteria.
- Several important ACMG/ENIGMA criteria remain non-automated and require
  manual review.
- Domain and interface annotations are incomplete and should not be treated as
  classification evidence.
- BS3 and PS3 require assay metadata to understand what biological mechanism was
  measured.
- The current priority score is heuristic. Thresholds and weights need
  additional sensitivity analysis and external justification before clinical or
  publication-level claims are made.
- A specific priority-score sensitivity test is now indicated: direct
  functional `PS3` evidence has 25 priority points, while high SpliceAI has 30.
  The current weights can place many functionally LOF `PS3+PM2_Supporting` VUS
  at the tier3 floor even though they are one evidence item short in the action
  plan. A future version should separate review priority from
  distance-to-threshold explicitly.
- The neighborhood component is partially redundant with the generated
  rule-based map. It is useful as context and review triage, but it should not
  be described as independent evidence.
- The framework still needs validation against external curated classifications,
  such as ENIGMA, ClinVar high-review variants, EQA examples, or local manual
  review data.
- Public ClinVar and ClinGen/ENIGMA assertions are current clinical/laboratory
  context, not a clinical reference set. They should be used to guide manual inspection of
  evidence and submitter context.
- Findlay SGE provides a useful external functional axis only for a restricted
  BRCA1 subset; BRCA2 remains much less covered by comparable dense functional
  data in the current analysis.
- A practical validation step is still missing: a small review-yield pilot
  comparing tier1/tier2, tier3, and tier4/PM2-only variants. The relevant
  outcome is not automatic reclassification, but whether manual review finds
  actionable external evidence more often in the higher-priority groups.

## Conclusion

The strongest current use of the ARIANE Module 1 snapshot is an auditable
manual-review queue. It separates variant-level signals, neighborhood-driven
context, and evidence-conflict cases, without turning those signals into new
classification criteria. This gives curators a more systematic first pass
through thousands of BRCA1/2 VUS.

The most useful current output is a structured review queue for internal
curation and method development. It is not yet a validated prioritization
method. External validation against curated clinical assertions, such as
ENIGMA, ClinVar high-review variants, EQA examples, or local manual review
data, is required before claiming that tier1/tier2 variants are truly more
likely to be clinically resolvable or more useful to review first.

A realistic next validation would measure review yield: for example, manually
review 30 tier1/tier2 variants, 30 tier3 variants, and 30 tier4 or PM2-only
variants, then compare how often actionable external evidence is found.
